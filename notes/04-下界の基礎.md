# 下界の基礎

このノートは、バンディットにおける「下界」が何を意味するのかを、基礎から整理するための導入です。
上界のノートでは「あるアルゴリズムがどれくらい良いか」を見ましたが、下界では「どんなアルゴリズムでも避けられない代償は何か」を見ます。

## 上界と下界の違い

上界は、特定のアルゴリズム $A$ に対して

```math
\mathbb{E}[R_T(A)] \leq f(T)
```

のような形で書かれます。

一方、下界は

```math
\mathbb{E}[R_T(A)] \geq g(T)
```

という形で、「どれだけ工夫しても、これ以下には下がらない」という主張になります。

学習上重要なのは、

- 上界は「このアルゴリズムはここまで良い」
- 下界は「どのアルゴリズムでもここまでしか良くならない」

という役割の違いです。

両方が揃うと、問題の本質的な難しさが見えます。

## リグレットをもう一度見る

累積リグレットは

```math
R_T
=
\sum_{t=1}^T (\mu^* - \mu_{I_t})
=
\sum_{i:\Delta_i>0} \Delta_i \, N_i(T)
```

でした。

この式からすぐ分かることは、下界を出したければ

- 劣最適腕をどれだけ引かざるをえないか

を下から抑えればよい、ということです。

つまり、下界でも中心は $N_i(T)$ です。
上界では $\mathbb{E}[N_i(T)]$ を「多くてもこの程度」と抑えましたが、下界では逆に「少なくともこの程度は必要」と言います。

## なぜ下界が存在するのか

直感は単純です。

最適腕を知るには、各腕の平均報酬をある程度は見分けなければいけません。
しかし、平均が近い腕どうしは少し引いただけでは見分けがつきません。

たとえば 2 本腕で

- 腕 1 の平均が $\mu_1 = 0.5 + \Delta$
- 腕 2 の平均が $\mu_2 = 0.5$

だったとします。

このとき、本当に腕 1 が良いと判断したいなら、
腕 2 をほとんど引かずに済ませるわけにはいきません。
もし腕 2 を全く調べなければ、

- 「実は腕 2 の方が少し良い世界」

との区別ができないからです。

ここに、下界の本質があります。

## 下界の読み方

バンディットでよく出る下界には、主に二種類あります。

### 1. instance-dependent な下界

これは「問題インスタンス $\nu$ を固定したとき、その問題は少なくともどれくらい難しいか」を表します。

確率的 2 項報酬の設定では、典型的に、ある定数 $c>0$ を用いて

```math
\mathbb{E}[R_T]
\geq
c \sum_{i:\Delta_i>0} \frac{\log T}{\Delta_i}
```

や、より精密には KL ダイバージェンスを使って

```math
\liminf_{T \to \infty}
\frac{\mathbb{E}[R_T]}{\log T}
\geq
\sum_{i:\Delta_i>0}
\frac{\Delta_i}{d(\mu_i,\mu^*)}
```

のような形になります。

ここで $d(p,q)$ は Bernoulli 分布どうしの KL ダイバージェンス

```math
d(p,q)
:=
p \log \frac{p}{q}
+
(1-p)\log\frac{1-p}{1-q}
```

です。

この下界は、「ギャップが小さい腕ほど見分けにくい」ことを反映しています。

### 2. minimax な下界

これは「最悪の問題インスタンスでは、どんなアルゴリズムでも少なくともどれくらい苦しいか」を表します。

典型的には

```math
\inf_A \sup_{\nu} \mathbb{E}_{\nu}[R_T(A)]
\geq
c \sqrt{KT}
```

のような形です。

ここで

- $\inf_A$ は全アルゴリズムの中で最良のものを選んでも
- $\sup_{\nu}$ は最悪の問題をぶつけられると

という意味です。

instance-dependent な下界が「この問題はこれだけ難しい」なのに対し、
minimax 下界は「最悪ケースではここまで悪くなる」を言っています。

## どうして $\log T$ が出るのか

上界では $\log T$ が Hoeffding 不等式から出てきました。
下界では意味が少し違います。

下界での $\log T$ は、

- 時間が長くなるほど、アルゴリズムはますます間違えにくくならなければならない
- そのためには、世界を区別するための情報をより多く集めなければならない

という要求から出てきます。

ざっくり言えば、

- 時刻 $T$ までずっと大きな後悔をしないためには
- 「どちらの腕が良い世界か」をかなり高い確率で当て続ける必要があり
- そのためには $\log T$ オーダーの情報が必要

という構図です。

この「情報」の量を測るのが KL ダイバージェンスです。

## どうして KL が出るのか

平均の差 $\Delta_i$ だけでは、「どれだけ区別しやすいか」は完全には分かりません。

たとえば、平均が同じだけ離れていても、

- 分布の形が違えば
- 観測からの見分けやすさも違う

ことがあります。

そこで、二つの分布をどれだけ見分けやすいかを表す量として KL ダイバージェンスが自然に現れます。

感覚的には

- 1 回観測すると、世界を区別するための情報が KL ぶんだけ増える
- だから、$\log T$ ぶんの情報を集めるには、おおむね $\log T / \mathrm{KL}$ 回くらい観測が必要

となります。

これが

```math
\frac{\log T}{\mathrm{KL}}
```

型の下界の直感です。

## 下界でよく出る主張

下界のノートや論文で、よく見る主張を並べるとつぎのようになります。

### 劣最適腕の試行回数の下界

```math
\mathbb{E}[N_i(T)]
\geq
\text{少なくとも } \frac{\log T}{\mathrm{KL}} \text{ オーダー}
```

### リグレット下界

```math
\mathbb{E}[R_T]
\geq
\text{少なくとも } \sum_i \Delta_i \frac{\log T}{\mathrm{KL}_i}
```

### 最悪ケースの下界

```math
\mathbb{E}[R_T]
\geq
\text{少なくとも } \sqrt{KT} \text{ オーダー}
```

もちろん、ここでいう「少なくとも」は、任意のアルゴリズムに対する主張です。

## 上界と下界をどう見比べるか

上界と下界を比べるときは、つぎの点を見ると整理しやすいです。

- $T$ に対する依存は一致しているか
- $\Delta_i$ に対する依存は一致しているか
- KL のような情報量が入っているか
- worst-case と instance-dependent のどちらを比べているか

たとえば UCB1 の標準上界は

```math
\sum_i \frac{\log T}{\Delta_i}
```

型でした。

一方、Bernoulli の instance-dependent 下界は

```math
\sum_i \frac{\Delta_i \log T}{d(\mu_i,\mu^*)}
```

型です。

小さいギャップでは $d(\mu_i,\mu^*)$ は $\Delta_i^2$ に比例するので、
両者はオーダーとして整合しています。

## 下界はなぜ上界より難しく見えるのか

上界は「このアルゴリズムを解析する」ので、対象が一つです。
下界は「どんなアルゴリズムでも逃げられない」ことを言うので、対象がアルゴリズム全体になります。

このため下界では、

- 別の世界を作る
- その二つの世界を区別する難しさを測る
- 区別できないなら、どちらかの世界で必ず後悔する

という、情報理論や仮説検定に近い発想が使われます。
