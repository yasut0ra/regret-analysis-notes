# リグレット解析の典型パターン（確率的 $K$ 本腕バンディット / UCB 系）

このノートは、UCB 系アルゴリズムのリグレット解析で何を証明すればよいかを、典型的な流れとして整理したものです。
細かい定数を覚えるより、まずは「どこで集中不等式を使い、どうやって劣最適腕の試行回数を数えるか」を掴むことを目的にしています。

## 問題設定

- 腕は $i = 1, \dots, K$
- 腕 $i$ の報酬列を $X_{i,1}, X_{i,2}, \dots$ とし、各腕ごとに i.i.d.
- 報酬は $X_{i,t} \in [0,1]$
- 腕 $i$ の期待報酬を $\mu_i$
- 最適平均を $\mu^* = \max_i \mu_i$
- 最適腕を $i^* \in \arg\max_i \mu_i$
- ギャップを $\Delta_i = \mu^* - \mu_i$ とする

時刻 $t$ に選んだ腕を $I_t$、時刻 $T$ までに腕 $i$ を引いた回数を $N_i(T)$ と書くと、累積リグレットは

```math
\begin{aligned}
R_T
&= \sum_{t=1}^T (\mu^* - \mu_{I_t}) \\
&= \sum_{i:\Delta_i>0} \Delta_i \,\mathbb{E}[N_i(T)].
\end{aligned}
```

です。
したがって、解析の中心は「各劣最適腕 $i$ について $\mathbb{E}[N_i(T)]$ をどう上から抑えるか」にあります。

## まず押さえるべき見取り図

UCB 型の解析は、ほぼ毎回つぎの形になります。

1. 経験平均のズレを抑えるための信頼半径を定義する
2. その半径の中に全ての経験平均が入る「良い事象」を定義する
3. 良い事象の下では、十分回数を引いた劣最適腕はもう選ばれないことを示す
4. 残る回数を「閾値までの試行回数 + 悪い事象の寄与」に分解する
5. それをリグレット分解に代入する

証明の本体は 3. と 4. です。
1. と 2. はそのための準備、5. は仕上げです。

## UCB 型アルゴリズムの基本形

初期化として、各腕を少なくとも 1 回は引いておくとします。
そうすれば $N_i(t-1)=0$ によるゼロ除算を避けられます。

経験平均を $\hat{\mu}_i(s)$ とし、信頼半径を

```math
\mathrm{rad}(s,t) = \sqrt{\frac{c \log t}{2s}}
```

のように置きます。
すると UCB 指標は

```math
\mathrm{UCB}_i(t)
= \hat{\mu}_i(N_i(t-1)) + \mathrm{rad}(N_i(t-1), t)
```

となります。
UCB1 では典型的に $c=4$ 相当の形が現れ、最終的に $\log T / \Delta_i$ 型の上界が出ます。

## ステップ 1. 良い事象を定義する

時刻 $t$ における良い事象を

```math
E_t
:=
\bigcap_{i=1}^K
\bigcap_{s=1}^t
\left\{
\left| \hat{\mu}_i(s) - \mu_i \right|
\leq
\mathrm{rad}(s,t)
\right\}
```

と置きます。

この事象の意味は単純で、
「時刻 $t$ までに登場しうる全ての腕・全ての試行回数について、経験平均が真の平均から大きく外れていない」です。

Hoeffding 不等式と合併界を使えば、$\mathbb{P}(E_t^c)$ を小さくできます。
重要なのは、$\sum_{t=1}^{\infty} \mathbb{P}(E_t^c)$ が有限になるように設計することです。
そうすると悪い事象の総寄与を定数で吸収できます。

## ステップ 2. 良い事象の下で劣最適腕が選ばれる条件を導く

良い事象 $E_t$ の下では、任意の腕 $i$ について

```math
\hat{\mu}_i(N_i(t-1))
\leq
\mu_i + \mathrm{rad}(N_i(t-1), t)
```

が成り立ちます。
また最適腕 $i^*$ については

```math
\hat{\mu}_{i^*}(N_{i^*}(t-1))
\geq
\mu^* - \mathrm{rad}(N_{i^*}(t-1), t)
```

なので

```math
\mathrm{UCB}_{i^*}(t) \geq \mu^*
```

です。

ここで、もし時刻 $t$ に劣最適腕 $i$ が選ばれたなら

```math
\mathrm{UCB}_i(t) \geq \mathrm{UCB}_{i^*}(t)
```

でなければいけません。
したがって $E_t$ の下では

```math
\mu_i + 2 \mathrm{rad}(N_i(t-1), t)
\geq
\mu^*
```

となり、整理して

```math
2 \mathrm{rad}(N_i(t-1), t) \geq \Delta_i
```

を得ます。

この一行が解析の核心です。
言い換えると、劣最適腕がまだ選ばれるなら、その腕の信頼半径はまだギャップ $\Delta_i$ より大きい、ということです。

## ステップ 3. 試行回数の閾値を置く

つぎに、劣最適腕 $i$ ごとに

```math
m_i := \left\lceil \frac{C \log T}{\Delta_i^2} \right\rceil
```

という閾値を取ります。
ここで $C$ は半径の定数に応じて決まる十分大きな定数です。

すると $t \leq T$ と $N_i(t-1) \geq m_i$ から

```math
2 \mathrm{rad}(N_i(t-1), t)
\leq
2 \mathrm{rad}(m_i, T)
\leq
\Delta_i
```

が従うようにできます。
よって良い事象の下では、$m_i$ 回以上引かれた劣最適腕 $i$ はそれ以上選ばれません。

## ステップ 4. pull counting をする

上の議論から、

- 腕 $i$ は最初の $m_i$ 回までは引かれうる
- それ以降に引かれるなら、その時刻では悪い事象 $E_t^c$ が起きている

と言えます。
したがって

```math
N_i(T)
\leq
m_i
+
\sum_{t=1}^T \mathbf{1}\{ I_t = i \text{ and } E_t^c \}
```

です。
期待値を取ると

```math
\mathbb{E}[N_i(T)]
\leq
m_i + \sum_{t=1}^T \mathbb{P}(E_t^c)
```

を得ます。

もし $\mathbb{P}(E_t^c) \leq O(t^{-3})$ のように抑えられていれば

```math
\sum_{t=1}^{\infty} \mathbb{P}(E_t^c) = O(1)
```

なので、

```math
\mathbb{E}[N_i(T)]
=
O\left(\frac{\log T}{\Delta_i^2}\right) + O(1)
```

となります。

## ステップ 5. リグレット上界へ戻す

最後に

```math
R_T = \sum_{i:\Delta_i>0} \Delta_i \,\mathbb{E}[N_i(T)]
```

へ代入すると

```math
R_T
=
O\left(
\sum_{i:\Delta_i>0} \frac{\log T}{\Delta_i}
\right)
```

が得られます。

## 簡略証明と標準証明の違い

このノートでは、学習のしやすさを優先して

- まず「全腕・全試行回数」をまとめた良い事象 $E_t$ を置く
- その補集合 $E_t^c$ の総和を定数として吸収する

という簡略化した流れで説明しています。

このやり方の利点は、証明の型が非常に見えやすいことです。
一方で、定数項はやや粗くなります。

UCB1 の標準的な証明では、良い事象を大きく一つにまとめず、
時刻 $t$ に実際に比較される

- 最適腕の経験平均
- 劣最適腕の経験平均

だけを取り出して、「どちらかの経験平均が大きくズレているか」を直接数えます。
このやり方だと、各劣最適腕について

```math
\mathbb{E}[N_i(T)]
\leq
\frac{8 \log T}{\Delta_i^2} + 1 + \frac{\pi^2}{3}
```

のような、よりシャープな定数が出ます。

詳細は [notes/02-UCB1-詳細証明.md](02-UCB1-詳細証明.md) にまとめています。

## この型で毎回チェックするポイント

- 初期化が書かれているか
  各腕を 1 回ずつ引くなど、$N_i(t-1)=0$ を避ける前処理が必要です。
- 良い事象の定義が「全腕・全試行回数」に対して取れているか
  合併界をかける対象が抜けると、その後の議論が崩れます。
- 最適腕側の下界をちゃんと使っているか
  $\mathrm{UCB}_{i^*}(t) \geq \mu^*$ を使うのが重要です。
- 閾値 $m_i$ に $t \leq T$ をどう入れるかが明示されているか
  ここを曖昧にすると定数が飛びます。
- 最後に $\Delta_i$ を掛け戻しているか
  $\mathbb{E}[N_i(T)]$ の上界だけで止めず、必ずリグレットへ戻します。

## よくあるつまずき

- $\hat{\mu}_i(s)$ の添字 $s$ と、実際に代入する $N_i(t-1)$ が混ざる
- 良い事象の時刻添字 $t$ と、試行回数添字 $s$ が混ざる
- 「良い事象なら選ばれない」と「悪い事象なら必ず選ばれる」を取り違える
- $\Delta_i = 0$ の最適腕を劣最適腕と同じように数えてしまう

## まとめ

UCB 系のリグレット解析は、結局のところ

1. 集中不等式で UCB のズレを抑える
2. 劣最適腕が選ばれるなら半径がまだ大きいことを示す
3. その事実を使って劣最適腕の試行回数を数える

という三段階に整理できます。
式の見た目は変わっても、証明の骨格はほぼ同じです。
